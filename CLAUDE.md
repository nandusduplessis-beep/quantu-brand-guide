# Project Guidelines

## Communication Rules

- When referencing GitHub pages, PRs, branches, repos, or any web resource, ALWAYS provide the full clickable URL — never just a name or path the user has to manually assemble.
- When giving instructions that involve navigating to a website, include the direct hyperlink every time.
- When giving step-by-step instructions, be precise about what the user needs to click or do. Say "Click Create pull request" not "Click Merge pull request" if the PR hasn't been created yet. Never skip steps or assume the user knows the GitHub workflow.
- Use exact button names and label them clearly. Don't conflate different steps (e.g., creating a PR vs merging a PR).
